# Traffic Light V2.1.1 使用教程

这份教程用于教别人使用 `Traffic Light_V2.1.exe` 和 `Traffic Light_V2.1_免安装测试版.exe`。它的工作流程是：

1. 选择本地视频文件，或填写实时视频流地址。
2. 程序调用 FFmpeg 按当前抽帧规则生成图片，可按间隔、确切时间点或指定时间段抽帧。
3. 程序监听图片目录，把新图片发送给大模型接口。
4. 大模型返回图片分析结果，结果显示在“实时分析结果”区域。
5. 如果开启“完成后删除图片”，处理完成的图片会自动删除。

## 二、第一次使用前检查

使用前确认三件事：

1. 电脑上可以运行 `Traffic Light_V2.1_免安装测试版.exe` 或 `Traffic Light_V2.1.exe`。
2. 发布包内存在 `tools\ffmpeg.exe`。
3. 如果模型需要通过跳板机访问，要保证 SSH 可用，且网络能连到跳板机。

首次使用建议先点“检查配置”。正常情况下至少应看到：

```text
FFmpeg 可用
SSH 可用
图片目录、结果目录、日志目录可写
```

如果换电脑使用，建议发送完整发布包或免安装测试包，不要只复制单个 exe。

## 三、日常使用流程

### 1. 打开 App

双击：

```text
01_免安装测试版\Traffic Light_V2.1_免安装测试版.exe
```

打开后会看到这些区域：

```text
顶部“下一步”
任务工作台
任务记录
模型连接
参数设置
运行日志
```

日常使用全部在“任务工作台”完成，包括模型连接、视频来源、分析规则和任务启动。“任务记录”用于查看历史任务和打开结果；完整连接参数放在“模型连接”；图片目录、结果目录、并发、超时和视频流稳定性参数放在“参数设置”；FFmpeg 输出和排错信息在“运行日志”。

顶部“下一步”会根据配置说明当前必须完成的操作。点击“查看指引”可查看标准顺序，点击“检查配置”会检查视频源、模型接口、提示词、FFmpeg、RTSP 安全接入和稳定性参数。

### 2. 选择视频输入来源

在“任务工作台”的“2. 视频来源”区域选择来源：

```text
本地视频文件
实时视频流
```

如果分析本地视频，点击“选择视频”，选择要分析的视频文件。

例如：

```text
视频所在文件夹\test.mp4
```

视频可以放在任意位置，不一定要放在程序数据目录里。

如果分析实时视频流，选择“实时视频流”，按下面顺序操作：

```text
1. 直接把拿到的视频流播放地址粘贴到“视频流地址”。
2. 点击“自动识别”或“校验格式”。
3. App 会自动判断是 RTSP/RTSPS、HLS、HTTP-FLV、RTMP、SRT、RTP/UDP 等类型。
4. 如果没有真实地址，也可以在“地址类型”里选一种，再点“填入该格式示例”。
5. 把示例里的 IP、端口、设备编号、通道编号改成真实信息。
6. 如果 RTSP 摄像头需要账号密码或 Token，不建议写在地址栏里；把地址填成 `rtsp://摄像头IP:554/stream1`，再到“安全接入 / 加密RTSP”里填写账号、密码或 Token。
7. 如果平台明确支持 RTSPS/TLS，再勾选“使用RTSPS/TLS”；普通 RTSP 账号密码认证不要勾选。
```

如果用户只拿到 `192.168.1.64:554/Streaming/Channels/101`、`example.com/live/test.flv` 或
`example.com/live/test.m3u8` 这类缺少 `rtsp://`、`http://` 的地址，可以直接粘贴后点“自动识别”。
软件会按常见规则自动补全协议；如果地址本身比较特殊，再手动补完整协议。

常见格式：

```text
rtsp://摄像头IP:554/stream1
rtmp://服务器地址/live/stream
http://服务器地址/live.m3u8
https://服务器地址/live.m3u8
http://服务器地址/live/camera01.flv
srt://服务器地址:9000?mode=caller
rtp://组播地址:端口
udp://组播地址:端口
```

加密 RTSP 说明：

```text
软件支持标准 RTSP 账号密码、Token、RTSPS/TLS，并会把安全播放地址交给 FFmpeg 抽帧。
日志和弹窗会自动隐藏密码。
如果摄像头是厂家私有加密码流，通用 FFmpeg 不能直接破解。
这种情况需要厂家 SDK、解密密钥，或让国标平台/视频网关转成标准 RTSP/RTSPS、HTTP-FLV、HLS 后再接入。
```

国标 GB28181 说明：

```text
GB28181 本身是 SIP 信令体系，不能把 gb28181:// 或 sip:// 地址直接填进 App。
正确做法是先在国标平台、视频网关或流媒体服务里生成可播放地址，
再把转出的 RTSP、HTTP-FLV、HLS(m3u8)、RTP、UDP 或 SRT 地址填入 App。
```

常见国标平台转流示例：

```text
rtsp://国标平台IP:554/设备国标编号/通道编号
http://国标平台IP:端口/rtp/设备国标编号_通道编号.live.flv
http://国标平台IP:端口/rtp/设备国标编号_通道编号/hls.m3u8
```

实时流默认是“完整分析不丢帧、断线自动重连”。如果大模型分析速度慢于抽帧速度，程序会保留已经抽到的帧并提示队列积压；只有在“参数设置 > 视频流稳定性”里主动勾选“低延迟优先（队列满时丢旧帧）”，才会牺牲完整性优先分析最新画面。

窗口缩放说明：

```text
软件主窗口支持放大、缩小和最大化。
窗口变化结束后，工作台分区、视频预览区和结果区会按比例重新摆放。
分析规则页在小窗口下会显示独立滚动区域，页面切换时不会重新创建顶层滚动画布。
```

### 3. 视频预览与抽帧取时

V2.1 新增任务工作台内嵌“视频预览与抽帧取时”区域，用于先看画面，再决定抽帧时间。这个区域不会直接开始分析，只负责辅助选择时间并把时间写回分析规则。

本地视频使用方式：

```text
1. 在“2. 视频来源”选择“本地视频文件”。
2. 点击“选择视频”选择本地视频。
3. 在右侧“视频预览与抽帧取时”区域点击“打开预览”。
4. 在预览区中点击播放，或拖动进度条定位画面。
5. 右侧时间会显示当前播放位置，格式为 HH:MM:SS。
6. 需要只抽某一帧时，定位到对应画面后点击“设为时间点”，再点击“应用时间点”。
7. 需要分析一段视频时，分别定位到开始画面和结束画面，点击“设为开始”“设为结束”，再点击“应用时间段”。
8. 应用后回到“分析规则”，确认抽帧模式和抽帧间隔即可。
```

实时视频流使用方式：

```text
1. 在“2. 视频来源”选择“实时视频流”，填写视频流地址和必要的账号密码、Token 或 TLS 选项。
2. 在右侧“视频预览与抽帧取时”区域点击“打开预览”。
3. 软件会使用当前取流配置打开实时画面。
4. 实时流没有固定总时长，预览区显示的是从打开预览开始的真实时间。
5. 看到需要分析的位置时，点击“设为时间点”或记录开始、结束时间。
6. 应用到分析规则后，再点击“开始分析”执行正式抽帧。
```

使用建议：

```text
本地视频：先用进度条找到关键画面，再选择时间点或时间段。
实时流：先确认画面能正常打开，再记录相对时间；如果实际任务需要长期运行，仍建议使用“按间隔抽帧”。
时间段模式：需要同时设置开始时间、结束时间和抽帧秒数，程序会在这段范围内按间隔抽帧。
```

### 4. 设置图片目录

“图片目录”是 FFmpeg 抽帧图片保存的位置，也是程序监听的位置。

默认是：

```text
Traffic Light_V2.1_数据\frames
```

一般不用改。

### 5. 检查接口

点击“测试模型连接”。

正常情况会提示：

```text
接口测试通过
可用模型：Qwen3.5-27B
```

如果提示接口失败，不要直接开始分析，先看本文“常见问题”部分。

### 6. 设置提示词

提示词决定大模型如何分析图片。

新版 App 内置了多种提示词模板，例如：

```text
通用详细报告
现场巡检报告
安全生产隐患
交通道路分析
施工现场分析
无人机视角分析
安防监控摘要
仓储物流盘点
设备仪表读数
文字与标识提取
质量缺陷检查
人员行为识别
事件快速摘要
```

使用方式：

```text
1. 在“提示词模板”下拉框选择合适模板。
2. 如果没有合适模板，选择“无（自行填写）”。
3. 在“图像分析提示词”里写自己的分析目标。
4. 在“我的模板名称”里填写一个容易识别的名字，例如“工地安全巡检”。
5. 点击“SAVE 保存为模板”。
6. 保存后的模板会以“我的模板：xxx”显示，下次打开软件仍然可选。
7. 只允许删除“我的模板”，系统内置模板不会被误删。
8. 提示词模板下拉框不会响应鼠标滚轮，避免误触切换模板。
```

提示词不要写得过泛。建议说明任务视角、关注对象、输出结构和不确定时的处理方式。例如：

```text
请以安全巡检人员视角分析这张图片，重点关注人员防护、设备状态、通道占用、火灾隐患和异常行为。
按“场景概况、风险点、风险等级、处理建议、结论”输出。
无法从图片确认的内容请写“不确定”，不要编造。
```

### 7. 设置抽帧规则

“抽帧模式”用于决定从视频中取哪些画面：

```text
按间隔连续抽帧：从视频开始或实时流当前时刻开始，按抽帧秒数持续抽帧。
确切时间点抽帧：只抽指定时间点附近的一张图。
时间段按间隔抽帧：只在开始时间到结束时间之间，按抽帧秒数抽帧。
```

“抽帧秒数”表示每隔多少秒抽一张图。确切时间点模式通常不需要调小间隔；时间段模式下，间隔越小，生成图片越多。

常用建议：

```text
短视频测试：5 或 10
普通巡检：10
长视频粗略分析：30 或 60
需要更细分析：1 到 3
```

数值越小，图片越多，分析越慢，接口压力越大。

### 8. 开始分析

点击“开始分析”。

程序会自动做三件事：

1. 如果当前选择的是“SSH 跳板机”路线，先自动启动 SSH 隧道。
2. 启动图片目录监听。
3. 调用 FFmpeg 从视频中抽帧。

正常日志大概会出现：

```text
启动 SSH 隧道
SSH 隧道 已启动
已启动监听：Traffic Light_V2.1_数据\frames
启动 FFmpeg：每 10 秒抽 1 帧
新图片入队
开始分析
分析完成
```

### 8. 查看结果

分析结果会直接显示在“开始分析”页的“实时分析结果”区域。

同时，程序会自动把结果保存为 Markdown 文件，默认目录是：

```text
Traffic Light_V2.1_数据\results
```

点击“打开结果文件夹”可以直接查看保存好的结果文件。

如果开启了“完成后删除图片”，分析成功后图片会自动删除，避免目录越来越大。

实时视频流开始前，程序会先尝试读取 1 帧做播放验证。RTSP 已支持 TCP 模式；默认“自动”会先使用 TCP，正式拉流首帧失败后自动切换 UDP 再试。也可以进入“参数设置 > 视频流稳定性”，把“RTSP传输”固定为 TCP 或 UDP。RTSP 如果三种探测方式只是超时，没有返回鉴权、404 或协议错误，程序会继续进入正式持续抽帧，并监控是否生成首帧；这样可以兼容首帧慢、关键帧间隔长的摄像头或国标平台。若正式启动后一段时间仍没有任何图片生成，程序会自动停止并给出诊断。RTMP 的 1935 端口在公网环境经常被限制，能选择时优先使用 RTSP、HTTP-FLV 或 HLS(m3u8) 播放地址。

本地视频文件分析完成后，程序会等待所有已抽出的图片处理完毕，然后自动停止后台监听。所有图片都成功时，右上角状态会回到“就绪”绿灯；如果全部失败、抽帧异常或重试耗尽，状态会自动切到“已停止”。

新版会在 FFmpeg 抽帧结束后再次扫描本批次生成的图片进行对账。即使 Windows 文件监听事件延迟，遗漏图片也会补充进入分析队列；只有目录稳定、队列为空、所有本批次图片都已成功或失败后，任务才会自动收尾。

如果勾选“完成后删除图片”，分析成功的图片会自动删除并在文件占用时重试；分析失败的图片会保留，便于检查接口返回、模型能力、图片损坏或网络问题。日志会明确说明保留的失败图片数量。

手动点击“停止”或运行中关闭窗口时，程序会先停掉 FFmpeg、监听器、SSH 隧道和排队任务，再删除当前 FFmpeg 抽帧批次里尚未分析完成的剩余图片，包含快速首帧图片。这个清理只匹配当前批次文件名前缀，不会删除历史批次、手工放入目录的图片或其他无关文件。若取消勾选“完成后删除图片”，停止时会按设置保留当前批次图片。

右上角状态灯使用强区分的红、黄、绿三色：就绪为稳定绿灯，异常或停止为稳定红灯，运行、监听、分析、测试、重试和收尾为平滑渐变的黄色呼吸灯。

### 9. 停止任务

实时视频流和“监听图片目录”属于持续运行任务，需要点击“停止任务”。

如果要关闭软件，建议先点“停止”，再关闭窗口。对于本地视频文件，任务完成后通常不需要手动停止；如果你中途停止本地视频抽帧，并且已勾选“完成后删除图片”，本次抽帧批次剩余图片会被清理掉。


不抽视频，只监听图片目录。适合你已经用别的程序把图片放进目录的情况。

停止监听、FFmpeg 和 SSH 隧道。

```text
测试模型连接
```

测试本地端口是否能连通，并读取服务端可用模型列表。换服务器后必须先点这个。


只清空界面日志，不影响文件和配置。

```text
运行本机诊断
```

检查 FFmpeg、SSH、图片目录、结果目录、当前接口地址和视频输入来源。遇到连接失败、无法抽帧或换电脑运行时，建议先点这个。

```text
任务工作台“模型服务”区域里的三条连接路线
```

公网大模型只填写公网接口和 API Key；私有化直连只填写本机能访问的内网接口；SSH 跳板机路线才填写 SSH 信息并自动启动隧道。

```text
处理已有图片
```

启动后会把图片目录里已经存在的 jpg/png/jpeg 也加入分析队列。

```text
完成后删除图片
```

图片分析成功后自动删除。测试阶段可以取消勾选，方便检查抽出来的图片。手动停止时，此选项也决定是否清理当前抽帧批次里还没分析完的剩余图片。

## 五、当前服务器配置说明

下面是 SSH 跳板机路线的通用示例，不包含任何真实服务器或密钥：

```json
{
  "api_url": "http://localhost:8080/v1/chat/completions",
  "api_key": "",
  "model": "qwen3-vl-plus",
  "auto_start_tunnel": true,
  "ssh_tunnel_command": "ssh -N -o ServerAliveInterval=60 -L 8080:10.0.0.20:8000 -p 22 user@jump.example.com"
}
```

这表示：

```text
本机访问地址：
http://localhost:8080/v1/chat/completions

本机 8080 端口通过 SSH 转发到：
10.0.0.20:8000

SSH 登录目标：
user@jump.example.com

SSH 端口：
22

请求模型名：
qwen3-vl-plus
```

注意：示例地址和账号必须替换成你自己的真实环境。程序真正发送给大模型的模型名以 `model` 字段为准。

## 六、更换服务器配置

更换服务器时，优先在 App 界面里改；也可以直接改 `stream_config.json`。

任务工作台的模型服务分成三条路线：

```text
路线一：公网大模型
直接访问 DashScope、OpenAI兼容接口、硅基流动等公网服务。不需要 SSH。

路线二：私有化部署大模型（SSH跳板机）
模型服务在内网或远程服务器后面，需要通过 SSH 跳板机转发。本路线会自动启动 SSH 隧道。

路线三：私有化部署大模型（直连）
电脑本身已经能访问模型服务所在内网或局域网。不需要 SSH。
```

请在“任务工作台”的“1. 模型服务”里选对路线。App 会自动只显示当前路线需要填写的表单，其它路线会隐藏。

### 情况 A：公网大模型

例如 DashScope 兼容接口是：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
```

操作步骤：

1. 打开 App。
2. 在“任务工作台”的“模型服务”选择“公网大模型”。
3. 点击 DashScope、OpenAI兼容或硅基流动示例按钮，或手动填写公网兼容接口。
4. 填写 API 密钥。
5. 点击“测试连接并读取模型”。
6. 如果 App 提示可用模型，会自动选择合适的视觉模型。
7. 在同一工作台继续填写视频来源和分析规则，再点击“开始分析”。

### 情况 B：私有化部署可以直连，不需要 SSH 隧道

例如新服务器接口是：

```text
http://192.168.1.50:8000/v1/chat/completions
```

操作步骤：

1. 打开 App。
2. 在“任务工作台”的“模型服务”选择“私有化直连”。
3. 可以点击“内网服务器示例”“本机模型示例”或“整理为标准接口路径”，再改成真实服务器地址。
4. 如果服务需要密钥，就填写 API 密钥；不需要密钥可以留空。
5. 点击“测试连接并读取模型”。
6. 看到可用模型后，在同一工作台继续配置并开始分析。

对应配置文件示例：

```json
{
  "api_url": "http://192.168.1.50:8000/v1/chat/completions",
  "api_key": "你的新密钥",
  "model": "服务端实际返回的模型名",
  "connection_mode": "private_direct",
  "auto_start_tunnel": false
}
```

### 情况 C：新服务器需要 SSH 隧道

App 不需要手写 SSH 命令。在“任务工作台”的“模型服务”选择：

```text
私有化部署：通过 SSH 跳板机访问内网模型服务
```

然后按表单填写：

```text
SSH服务器：跳板机公网 IP 或域名，例如 1.2.3.4
SSH端口：SSH 登录端口，例如 22 或 56789
用户名：SSH 用户名，例如 root 或 ubuntu
私钥文件：如果用密钥登录，就选择 .pem/.key/id_rsa 文件；不用密钥可以留空
本机端口：本机转发端口，通常填 8080
模型服务地址：远端服务器能访问到的模型服务地址，例如 10.0.0.8 或 127.0.0.1
服务端口：模型服务端口，例如 8000 或 8072
接口路径：模型接口路径，例如 /v1/chat/completions
```

填完后点击：

```text
生成本机接口地址
```

App 会自动把“接口地址”改成类似：

```text
http://localhost:8080/v1/chat/completions
```

并自动生成 SSH 命令预览。普通用户不需要复制或修改这条命令。

操作步骤：

1. 打开 App。
2. 在“任务工作台”的“模型服务”选择 SSH 跳板机。
3. 选择“私有化部署：通过 SSH 跳板机访问内网模型服务”。
4. 填写 SSH 登录信息和远端模型服务信息。
5. 点击“生成本机接口地址”。
6. 填写 API 密钥。
7. 点击“测试SSH并读取模型”。
8. 看到可用模型后，在同一工作台继续配置并点击“开始分析”。

如果 SSH 登录需要输入密码，勾选：

```text
需要输密码时打开SSH窗口
```

这样启动隧道时会弹出一个 SSH 窗口，用户可以在窗口里输入密码。更推荐使用私钥文件，这样最省心。

对应配置文件示例：

```json
{
  "api_url": "http://localhost:8080/v1/chat/completions",
  "api_key": "你的新密钥",
  "model": "服务端实际返回的模型名",
  "connection_mode": "private_ssh",
  "auto_start_tunnel": true,
  "ssh_host": "1.2.3.4",
  "ssh_port": 22,
  "ssh_user": "ubuntu",
  "ssh_key_path": "",
  "ssh_local_port": 8080,
  "ssh_remote_host": "10.0.0.8",
  "ssh_remote_port": 8000,
  "ssh_api_path": "/v1/chat/completions"
}
```

注意：只有 SSH 跳板机路线会自动启动 SSH 隧道。公网路线和私有化直连路线都不会启动 SSH。不要把 `localhost:8080` 的接口留在公网路线或直连路线里，除非本机真的运行了模型服务。

### 情况 D：服务器接口带自定义路径

有些服务不是标准：

```text
/v1/chat/completions
```

而是类似当前项目：

```text

```

这种情况下，接口就必须完整填写：

```text
http://localhost:8080/qwen3.6-27b/v1/chat/completions
```

程序会自动尝试读取：

```text
http://localhost:8080/qwen3.6-27b/v1/models
```

如果能读到模型列表，就会自动校验模型名。

## 八、配置文件字段说明

配置文件路径：

```text
Traffic Light_V2.1_数据\config\stream_config.json
```

每次在 App 里保存设置时，程序会先生成一份备份：

```text
stream_config.json.bak
```

如果误改了配置，可以参考这个备份文件恢复。

字段说明：

```text
image_dir
抽帧图片保存和监听目录。

source_type
视频输入来源。file 表示本地视频文件，stream 表示实时视频流。

video_file
本地视频文件路径。

stream_url
实时视频流地址，例如 rtsp:// 或 http:// 地址。界面中“自动识别”可对常见 IP:554、.flv、.m3u8 地址自动补全协议。

stream_format
实时视频流格式选择。默认是“自动识别”，普通用户不需要手动修改。

rtsp_username / rtsp_password
RTSP/RTSPS 标准账号、密码或 Token。建议在界面“安全接入 / 加密RTSP”里填写，不要直接写进地址栏。

rtsp_use_tls
是否使用 RTSPS/TLS。只有摄像头、国标平台或视频网关明确支持 RTSPS/TLS 时才开启。

connection_mode
连接路线。public 表示公网大模型，private_ssh 表示 SSH 跳板机私有化，private_direct 表示私有化直连。

api_key
接口密钥。

model
服务端实际可用的模型 ID。

prompt
每张图片发送给大模型时使用的分析提示词。

selected_prompt_preset
当前选择的提示词模板名称。选择“无（自行填写）”时，用户需要手动填写 prompt。

custom_prompt_templates
用户自己保存的提示词模板。App 会自动维护，普通用户不需要手动编辑。

max_retries
请求失败后的最大重试次数。

max_image_size
发送给模型前压缩图片的最长边尺寸。

concurrency
并发分析线程数。一般保持 1，避免压垮模型服务。

max_tokens
模型最多输出 token 数。越大结果越长。

temperature
模型随机性。0.2 到 0.5 比较稳。

request_timeout
单次接口请求超时时间，单位秒。

delete_processed
分析成功后是否删除图片。

process_existing
启动后是否处理目录里已有图片。

frame_interval
FFmpeg 每隔多少秒抽一张图。

ffmpeg_low_cpu
是否启用低CPU抽帧。默认启用，适合普通电脑和长视频，能降低抽帧时的 CPU 峰值。

ffmpeg_threads
FFmpeg 抽帧线程数。默认 1，更稳、更省 CPU；如果电脑性能强、希望本地视频抽帧更快，可以调到 2 到 4。

auto_start_tunnel
是否自动启动 SSH 隧道。新版界面会根据连接路线自动设置，普通用户不用手动改。

ssh_tunnel_command
SSH 端口转发命令。
```

## 九、常见问题

### 1. 日志显示 `API 异常：状态码 404`

常见原因：

```text
接口路径写错
模型名写错
服务器没有这个接口
```

解决方法：

1. 点击“测试模型连接”。
2. 看 App 是否能读出可用模型。
3. 如果读出的模型和当前模型不一致，App 会自动改成服务端模型。
4. 如果还是 404，重点检查 `api_url` 路径。

DashScope 兼容模式必须使用完整地址：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
```

如果只填到：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1
```

新版 App 会自动补全，但旧版会在真正分析时 404。

图片分析必须选择视觉模型，优先使用：

```text
qwen3-vl-plus
qwen3-vl-flash
qwen-vl-max
qwen-vl-plus
```

不要用普通文本模型做图片分析，例如 `qwen3.7-max`、`qwen3.5-plus` 这类模型通常不适合接收图片。

### 2. 日志显示 `localhost:8080 无法连接`

说明本机 8080 端口没有服务。

常见原因：

```text
SSH 隧道没有启动
SSH 命令写错
服务器地址或端口写错
远程服务没启动
```

解决方法：

1. 确认连接路线选择了“私有化部署：通过 SSH 跳板机访问内网模型服务”。
2. 检查 SSH服务器、用户名、模型服务地址、服务端口和接口路径。
3. 点击“测试SSH并读取模型”。

### 3. FFmpeg 启动后很快退出

如果日志显示：

```text
FFmpeg 已退出，退出码 0
```

通常是正常的，表示视频抽帧完成了。

如果退出码不是 0，常见原因是视频路径错误、视频损坏、FFmpeg 不支持该格式，或者实时视频流地址不可访问。

### 4. 实时视频流无法抽帧

常见原因：

```text
视频流地址没有带协议，且自动识别无法判断具体协议
摄像头用户名或密码错误
电脑网络无法访问摄像头或流媒体服务器
RTSP/RTMP 服务被防火墙拦截
视频流本身没有视频轨道
```

解决方法：

1. 先确认地址能被 VLC 或其他播放器打开。
2. RTSP 地址优先保持为 `rtsp://摄像头IP:554/stream1` 这类播放地址，账号密码或 Token 填到“安全接入 / 加密RTSP”；如果只有 IP:554 地址，可以先粘贴后点“自动识别”。
3. 抽帧秒数先设为 5 或 10，确认稳定后再调小。
4. 查看“运行日志”里的 FFmpeg 报错。

### 5. 只抽出 1 张图片

这是正常现象之一。

例如一个 12 秒视频，抽帧间隔设置为 10 秒，就只会抽出 1 张左右。

如果想多分析几张，把“抽帧秒数”改小，例如：

```text
1
2
5
```

### 6. 图片已经生成，但没有分析结果

检查：

```text
是否点击了“监听图片目录”或“开始分析”
图片目录是否和 App 里的“图片目录”一致
接口测试是否通过
日志里是否有 API 错误
```

### 7. 结果太短

可以调大：

```text
输出字数
```

也可以把提示词写得更明确，例如要求：

```text
请按“画面概述、关键目标、异常风险、结论”四部分详细分析。
```

### 8. 结果太慢

可以：

```text
增大抽帧秒数
减小输出字数
保持并发为 1
减小图片上限，例如 720
```

如果远端模型算力很强，再考虑把并发调到 2 或 3。

```text
updates
```

更新功能不会覆盖用户的接口密钥、任务记录、分析结果和抽帧图片。下载异常、网络中断、校验失败或用户取消下载时，软件会给出明确提示，并清理未完成的临时下载文件。

## 十三、V2.1 分析报告中的抽帧图片

V2.1 的分析报告会把每次送入模型分析的抽帧图片一起写入报告材料，方便评审、复盘和归档时直接对照“图片”和“模型分析结果”。

当任务开始后，软件会在结果目录中生成：

```text
analysis_时间戳.md
analysis_时间戳.jsonl
analysis_时间戳_assets
```

其中：

```text
analysis_时间戳.md       人可以直接阅读的分析报告
analysis_时间戳.jsonl    结构化结果记录，便于后续程序处理
analysis_时间戳_assets   报告中引用的抽帧图片目录
```

报告中的每条分析记录会尽量包含：

```text
抽帧图片
抽帧时间
输入来源
分析序号
分析完成时间
模型返回内容
```

如果任务停止、临时图片被清理或源图片已经不存在，报告不会崩溃中断，而是写入“对应抽帧图片缺失”的说明，日志中也会留下可追踪记录。

归档或发送报告时，请同时保留 `.md` 文件和同名 `_assets` 目录。只复制 `.md` 文件会导致报告里的图片无法显示。
